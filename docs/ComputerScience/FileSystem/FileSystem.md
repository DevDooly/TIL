# File System (파일 시스템)

파일 시스템(File System)은 컴퓨터가 저장 매체(HDD, SSD 등)에 데이터를 저장하고, 검색하고, 관리하는 방법을 정의한 체계입니다. 데이터는 '파일'이라는 단위로 저장되며, 이 파일들은 '디렉토리(폴더)'를 통해 계층적으로 정리됩니다.

## 1. 주요 역할

*   **저장 및 검색:** 데이터를 물리적 위치에 기록하고, 필요할 때 빠르게 찾아올 수 있도록 주소를 관리합니다.
*   **계층 구조 관리:** 파일과 폴더의 트리 구조를 유지하여 사용자가 직관적으로 데이터를 관리할 수 있게 합니다.
*   **권한 및 보안:** 파일의 소유자, 접근 권한(읽기/쓰기/실행)을 관리하여 보안을 유지합니다.
*   **메타데이터 관리:** 파일 이름, 생성일, 수정일, 크기 등의 부가 정보를 관리합니다.
*   **무결성 유지:** 시스템 충돌 시 데이터 손상을 방지하거나 복구하는 메커니즘(저널링 등)을 제공합니다.

## 2. 주요 파일 시스템 종류

### Windows
*   **FAT32 (File Allocation Table 32):** 호환성이 매우 좋아 Windows, Mac, Linux, 게임기 등 대부분의 기기에서 인식됩니다. 하지만 단일 파일 크기가 4GB로 제한되는 단점이 있습니다.
*   **NTFS (New Technology File System):** Windows의 표준 파일 시스템입니다. 대용량 파일 지원, 권한 관리, 압축, 암호화 등 고급 기능을 제공하지만, Mac에서는 기본적으로 읽기만 가능합니다.
*   **exFAT (Extended FAT):** FAT32의 4GB 파일 크기 제한을 없앤 버전입니다. Windows와 Mac 모두에서 읽기/쓰기가 가능하여 외장 하드나 USB 드라이브에 많이 사용됩니다.

### macOS
*   **HFS+ (Mac OS Extended):** 과거 macOS의 표준 파일 시스템이었습니다.
*   **APFS (Apple File System):** 최신 macOS, iOS, iPadOS 등을 위해 설계된 파일 시스템으로, SSD에 최적화되어 있으며 암호화와 복제(Cloning) 속도가 매우 빠릅니다.

### Linux
*   **ext4 (Fourth Extended Filesystem):** 리눅스의 가장 대표적인 표준 파일 시스템입니다. 안정적이고 성능이 우수하며 대용량 볼륨을 지원합니다.
*   **XFS:** 대용량 파일 처리에 매우 뛰어난 성능을 보이는 고성능 64비트 저널링 파일 시스템입니다.
*   **Btrfs:** 스냅샷, RAID 지원 등 고급 기능을 갖춘 차세대 리눅스 파일 시스템입니다.

## 3. 관련 개념

*   **마운트 (Mount):** 물리적인 저장 장치를 디렉토리 트리의 특정 위치에 연결하여 운영체제가 인식하게 하는 과정입니다.
*   **파티션 (Partition):** 하나의 물리적 디스크를 논리적으로 나눈 구획입니다. 각 파티션마다 다른 파일 시스템을 적용할 수 있습니다.
*   **저널링 (Journaling):** 데이터를 쓰기 전에 변경 사항을 로그(저널)에 먼저 기록하는 기술입니다. 시스템이 비정상 종료되더라도 저널을 통해 빠르게 복구할 수 있습니다.
